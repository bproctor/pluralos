sort

Name


sort - Sort, merge, or sequence check text files

Synopsis


sort long-option
sort [-m][-o output][-bdfinru][-t char][-k keydef]... [file ...]
sort -c [-bdfinru][-t char][-k keydef][file]

Description


The sort utility sorts lines of the files specified and writes the result to the specified output or the standard output. sort can also perform the function of merging presorted file together and can check that a file is correctly sorted.

Options


-b Ignore any leading blanks on the line when sorting
-c Check that a single input file is sorted. No output is produced. The exit code will be EXIT_SUCCESS if the file is sorted and EXIT_FAILURE if not.
-d Specifies that only blanks and alphanumeric characters are relevent when sorting
-f Use case-insensitive comparisons when sorting
-i Ignore non-printable characters
-k keydef Specify the sort key field definition.
-m Merge only. The input file(s) are assumed to be already sorted.
-n Numeric sort. Restricts the sort key to an initial numeric string
-o output Specified the name of the output file to be output. If this option is not specified, the output is written to the standard output.
-r Sort in reverse order
-t char Use char as the field seperator character. If this option is not specified, blanks are used as field seperators.
-u Unique. Suppress all but one in each set of lines having equal keys. If this option is used with the -c option, then sort will check that there are no lines with duplicate keys in addition to checking whether the file is sorted or not.
--help Print the help message and exit
--version Print program version and exit
