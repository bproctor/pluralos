nice

Name


nice - Invoke a utility with an altered scheduling priority

Synopsis


nice long-option
nice [-n increment] utility [arg ...]

Description


The nice utility invokes the utility utility with an altered scheduling priority than the default. By default, Plural sets user-level programs to a priority of 16 (0-32, 0 being highest). If no increment is given to nice on the command line, 10 is assumed. This will set the default priority to 26. If the user lacks appropriate privileges to affect the priority as requested, nice will give a warning message, but will not prevent utility from executing.

The exit status of nice is the same as utility's exit status. If there is an error in the nice utility. The following may be returned:
1-125 An error occured in the nice utility
126 utility found, but could not be executed
127 utility could not be found

Options


-n increment Change the processes scheduling priority by increment. increment can be either positive or negative.
--help Print the help message and exit
--version Print program version and exit
